2.1 AI Agent 发展史

从简单的聊天机器人到能自主执行任务的智能体，AI Agent 经历了快速演进。了解这段发展历程，有助于我们理解 OpenClaw 在 Agent 技术浪潮中的定位。

AI Agent 从对话工具到执行助手的技术演进

2022年
ChatGPT 时代：对话式 AI 兴起，但只能"说话"不能"动手"
2023年
Function Calling：AI 开始能调用外部工具，但能力有限
2024年
Agent 框架爆发：AutoGPT、LangChain 等让 AI 能自主规划任务
2025-2026年
OpenClaw 时代：AI 真正成为"数字员工"，能操作电脑、自主执行
💡 关键转折点

从"对话"到"执行"是 AI Agent 的质变。OpenClaw 代表的新一代 Agent，不再满足于回答问题，而是能直接操作系统、调用工具、完成复杂任务流。

2.2 四层架构设计

OpenClaw 采用分层架构设计，从用户交互到模型调用，每一层职责清晰，确保系统的灵活性和可扩展性。

数据流：用户消息 → Channels → Gateway → Agent → Provider → 返回响应

1
Channels（渠道层）
连接 20+ 消息渠道（飞书、WhatsApp、Telegram等），让 AI 住在你的社交软件里
2
Gateway（网关层）
消息路由、认证授权、会话管理，中央控制平面
3
Agent（智能体层）
消息流入、身份、灵魂、工具、记忆、技能、路由与处理、思考与决策
4
Provider（模型层）
对接 Claude、OpenAI 等大模型，提供底层 AI 能力

Chapter 03 · 第三章

"龙虾"拎包入住 —— 从一键部署到"开门接客"

5分钟让你的小龙虾上岗。本章详解本地安装、Docker部署、云端部署三种方案，以及国内网络优化配置。

3.1 环境准备

部署 OpenClaw 前，请确保系统满足以下要求：

项目 要求
Node.js ≥ 18.0.0（推荐 20.x LTS）
npm 或 yarn npm ≥ 9.0 或 yarn ≥ 1.22
操作系统 macOS / Linux / Windows (WSL2)
内存 最低 2GB，推荐 4GB+
磁盘空间 500MB（不含模型和日志）
💡
检查 Node.js 版本：
node --version 或 npm --version

3.2 本地安装

macOS 安装（推荐 Homebrew）

macOS 用户推荐使用 Homebrew 安装，简单快捷。

bash
# 1. 安装 Homebrew（如未安装） /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)" # 2. 安装 Node.js 22 brew install node@22 # 3. 添加 Node 到 PATH echo 'export PATH="/opt/homebrew/opt/node@22/bin:$PATH"' >> ~/.zshrc source ~/.zshrc # 4. 全局安装 OpenClaw npm install -g @openclaw/cli # 5. 验证安装 openclaw --version # 6. 初始化工作区 openclaw init my-agent cd my-agent

Windows 安装（WSL2 方式）

Windows 用户强烈推荐使用 WSL2（Windows Subsystem for Linux），提供与 Linux/macOS 一致的体验。

步骤一：安装 WSL2

powershell
# 以管理员身份打开 PowerShell，运行： wsl --install # 重启电脑后，设置 Ubuntu 用户名密码 # 然后更新系统 sudo apt update && sudo apt upgrade -y

步骤二：在 WSL2 中安装 OpenClaw

bash
# 1. 安装 Node.js 22 curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash - sudo apt-get install -y nodejs # 2. 验证 Node 安装 node --version # v22.x.x npm --version # 10.x.x # 3. 全局安装 OpenClaw npm install -g @openclaw/cli # 4. 验证安装 openclaw --version # 5. 初始化工作区 openclaw init my-agent cd my-agent

Linux 安装（Ubuntu/Debian）

bash
# 1. 安装 Node.js 22 curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash - sudo apt-get install -y nodejs # 2. 全局安装 OpenClaw npm install -g @openclaw/cli # 3. 初始化工作区 openclaw init my-agent cd my-agent # 4. 启动 Gateway openclaw start

目录结构说明

初始化完成后，工作区目录结构如下：

bash
my-agent/ ├── .openclaw/ # 运行时数据 │ ├── gateway.pid # 进程ID │ ├── sessions.json # 会话存储 │ └── logs/ # 运行日志 ├── workspace/ # Agent 工作区 │ ├── SOUL.md # 智能体人格定义 │ ├── TOOLS.md # 工具配置 │ └── skills/ # 技能目录 ├── config.yaml # 主配置文件 └── package.json # 项目依赖
💡
Windows 用户注意：WSL2 是运行 OpenClaw 的最佳方式。相比原生 Windows，WSL2 提供更好的文件系统性能和兼容性，同时保持与 macOS/Linux 一致的开发体验。

3.3 Docker 部署（生产推荐）

Docker 部署提供隔离环境，适合生产服务器部署。

快速启动

bash
# 创建数据目录 mkdir -p ~/openclaw-data/{workspace,memory,logs} # 运行容器 docker run -d \ --name openclaw \ -p 127.0.0.1:18789:18789 \ -v ~/openclaw-data:/data \ -e ANTHROPIC_API_KEY=sk-ant-xxx \ -e TELEGRAM_BOT_TOKEN=your-token \ openclaw/openclaw:latest

Docker Compose 配置

bash
version: "3.8" services: openclaw: image: openclaw/openclaw:2026.3.8 container_name: openclaw-gateway restart: unless-stopped ports: - "127.0.0.1:18789:18789" volumes: - ./workspace:/data/workspace - ./memory:/data/memory - ./logs:/data/logs environment: ANTHROPIC_API_KEY: ${ANTHROPIC_API_KEY} OPENAI_API_KEY: ${OPENAI_API_KEY} deploy: resources: limits: memory: 2G

3.4 云端部署（国内优化）

针对国内网络环境优化的部署方案，避免国际网络不稳定问题。

阿里云 ECS 一键部署 > 前往部署

bash
# 使用阿里云容器镜像服务加速 docker run -d \ --name openclaw \ -p 127.0.0.1:18789:18789 \ -e ANTHROPIC_API_KEY=sk-ant-xxx \ -e DASHSCOPE_API_KEY=sk-xxx \ -e MODEL_DEFAULT_PROVIDER=dashscope \ -e MODEL_DEFAULT_MODEL=qwen-max \ registry.cn-hangzhou.aliyuncs.com/openclaw/openclaw:2026.3.8

腾讯云 Lighthouse 应用模板 > 前往部署

腾讯云已上架 OpenClaw 应用模板，3分钟完成部署：

  1. 进入腾讯云 Lighthouse 控制台
  2. 创建实例 → 选择"应用镜像" → 搜索"OpenClaw"
  3. 选择配置（推荐 2核4G 及以上）
  4. 创建后自动完成初始化

华为云 Flexus 应用服务器 > 前往部署

华为云提供 OpenClaw 专属云服务器，9.9元轻松"养虾"：

  1. 选购 Flexus 云服务器 L 实例
  2. 选用 OpenClaw 应用镜像
  3. 配置 AI 大模型与 API Key
  4. 配置 IM 助手，开启"养虾"

国内网络优化配置

bash
# config.yaml 国内优化版 models: default: "qwen-max" providers: - name: "dashscope" api_key: "${DASHSCOPE_API_KEY}" base_url: "https://dashscope.aliyuncs.com/api/v1" channels: telegram: enabled: false feishu: enabled: true

5.1 $1,100的恐怖故事

有用户反馈，Agent 进入死循环疯狂调用 API，一天烧掉 $1,100。

🚨 防炸秘籍
  1. 设置日预算上限 `maxCostPerDay`
  2. 开启执行确认 `approve` 模式
  3. 定期检查 `openclaw doctor`

5.2 权限隔离

  • 权限隔离：为 AI 申请独立的 Gmail、GitHub
  • 沙箱执行：敏感操作开启人工审核

5.3 警惕恶意技能
