Chapter 01 · 第一章

初识这只"神虾" —— 它是谁？能干啥？

OpenClaw 不是大模型，而是运行在你设备上的智能界面。ChatGPT 是"顾问"（只动口），OpenClaw 是"员工"（既动口又动手）。

1.1 别再把"龙虾"当聊天机器人

很多人第一次听说 OpenClaw 时，会误以为它是另一个 ChatGPT 或 Claude。但实际上，OpenClaw 的定位完全不同。它不是一个大模型，而是一个运行在你本地设备上的智能界面（Interface）。

这就像云游戏 vs 本地主机的区别：云游戏所有计算都在云端，画面通过网络传给你；而本地主机直接在你的设备上运行。OpenClaw 就是后者，它直接操作你的电脑，读取你的文件，运行你的程序。

1.2 "养虾"文化简史

龙虾（OpenClaw）最初是 Anthropic 的 Claude Code 的开源替代品。Claude Code 仅限于终端使用，而 OpenClaw 让它接入各种社交渠道——Telegram、WhatsApp、Discord、甚至飞书。

"Claude 是顾问，只动口；OpenClaw 是员工，既动口又动手。"

在短短 4 个月内，OpenClaw 的 GitHub Star 数突破 28 万，正式超越 React 成为全球第一的软件项目。

Chapter 02 · 第二章

解剖"小龙虾" —— 揭秘 Agent 运作的底层黑科技

理解 OpenClaw 的架构，就像了解一只龙虾的生理结构——知道哪里是大脑、哪里是手脚，才能更好地"饲养"它。

2.1 三层骨架架构

OpenClaw 采用解耦的架构设计，确保其灵活性。整个系统由三个核心组件组成：

1
Gateway（中央控制平面）
这是龙虾的大脑，负责 WebSocket 调度、维护会话 (Session) 和管理智能体。
2
Node（执行节点）
这是龙虾的四肢，负责具体执行任务，如调用摄像头、录屏、执行系统命令。
3
Channel（消息渠道）
这是龙虾的耳朵，连接各种即时通讯平台，如 Telegram、飞书、钉钉。

2.2 龙虾的"记忆术"

为了不让 AI 变成"鱼的记忆"，OpenClaw 设计了精密的四层记忆层级：

SOUL.md（不可变内核）
定义 Agent 的人格、价值观和核心身份
TOOLS/Skills（动态工具）
当前加载的功能模块
USER.md/MEMORY.md（持久记忆）
存储用户的偏好、历史决策和长期事实，支持语义搜索 (RAG)
Session（实时对话）
当前的实时语境，当对话过长时，系统会自动进行上下文压缩

2.3 黑科技：上下文压缩

当对话历史超过模型上下文窗口时，OpenClaw 会自动进行上下文压缩 (Context Compaction)，保留关键信息，丢弃无关内容。

🧠 压缩算法

使用语义相似度计算，保留与当前任务相关的对话历史，将长对话压缩为简洁的摘要。
