数据流：用户消息 → Channels → Gateway → Agent → Provider → 返回响应

1

3
Agent（智能体层）
消息流入、身份、灵魂、工具、记忆、技能、路由与处理、思考与决策
4
Provider（模型层）
对接 Claude、OpenAI 等大模型，提供底层 AI 能力

3.2 本地安装

macOS 安装（推荐 Homebrew）

macOS 用户推荐使用 Homebrew 安装，简单快捷。

bash
# 1. 安装 Homebrew（如未安装） /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)" # 2. 安装 Node.js 22 brew install node@22 # 3. 添加 Node 到 PATH echo 'export PATH="/opt/homebrew/opt/node@22/bin:$PATH"' >> ~/.zshrc source ~/.zshrc # 4. 全局安装 OpenClaw npm install -g @openclaw/cli # 5. 验证安装 openclaw --version # 6. 初始化工作区 openclaw init my-agent cd my-agent

Windows 安装（WSL2 方式）

Windows 用户强烈推荐使用 WSL2（Windows Subsystem for Linux），提供与 Linux/macOS 一致的体验。

步骤一：安装 WSL2

powershell
# 以管理员身份打开 PowerShell，运行： wsl --install # 重启电脑后，设置 Ubuntu 用户名密码 # 然后更新系统 sudo apt update && sudo apt upgrade -y

步骤二：在 WSL2 中安装 OpenClaw

bash
# 1. 安装 Node.js 22 curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash - sudo apt-get install -y nodejs # 2. 验证 Node 安装 node --version # v22.x.x npm --version # 10.x.x # 3. 全局安装 OpenClaw npm install -g @openclaw/cli # 4. 验证安装 openclaw --version # 5. 初始化工作区 openclaw init my-agent cd my-agent

Linux 安装（Ubuntu/Debian）

bash
# 1. 安装 Node.js 22 curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash - sudo apt-get install -y nodejs # 2. 全局安装 OpenClaw npm install -g @openclaw/cli # 3. 初始化工作区 openclaw init my-agent cd my-agent # 4. 启动 Gateway openclaw start

目录结构说明

初始化完成后，工作区目录结构如下：

bash
my-agent/ ├── .openclaw/ # 运行时数据 │ ├── gateway.pid # 进程ID │ ├── sessions.json # 会话存储 │ └── logs/ # 运行日志 ├── workspace/ # Agent 工作区 │ ├── SOUL.md # 智能体人格定义 │ ├── TOOLS.md # 工具配置 │ └── skills/ # 技能目录 ├── config.yaml # 主配置文件 └── package.json # 项目依赖

Windows 用户注意：WSL2 是运行 OpenClaw 的最佳方式。相比原生 Windows，WSL2 提供更好的文件系统性能和兼容性，同时保持与 macOS/Linux 一致的开发体验。

Chapter 04 · 第四章

"特种兵"训练 —— Skills 技能系统与自动化

Skill 是工作的 SOP（标准作业程序），不是代码，而是给 AI 的操作手册。官方技能市场有 24,756 技能随便挑。

4.1 Skill是什么？

📋 Skill = 工作的 SOP（标准作业程序）

不是代码，而是给 AI 的操作手册。告诉它：遇到什么情况、用什么工具、按什么步骤执行。

Skill 的核心组成

组件 作用 示例
SKILL.md 技能说明文档 定义技能用途、触发条件
Tools 可调用的工具 浏览器、文件操作、API调用
Prompts 提示词模板 标准化输出格式
Config 配置参数 API密钥、超时设置

Skill 工作流程

1
触发识别
AI 判断用户请求是否匹配该技能的触发条件
2
参数提取
从对话中提取所需参数（时间、地点、关键词等）
3
工具调用
按 SOP 步骤调用相应工具执行操作
4
结果输出
格式化返回结果，必要时执行后续操作

4.3 Skill开发教程

除了使用现成技能，你还可以开发自定义 Skill 来满足特定需求。以下是完整的开发指南。

第一步：创建Skill目录结构

bash
# 创建工作目录 mkdir -p ~/.openclaw/workspace/skills/my-skill cd ~/.openclaw/workspace/skills/my-skill # 创建必要文件 touch SKILL.md touch config.yaml mkdir -p scripts/

第二步：编写SKILL.md

markdown
# My Skill - 股票行情查询 ## 描述 查询指定股票的实时行情数据，包括价格、涨跌幅、成交量等信息。 ## 触发条件 当用户询问以下类型的问题时触发： - "查询[股票代码]的行情" - "[股票代码]现在多少钱" - "看看[股票名称]的走势" ## 参数 - symbol: 股票代码（如：AAPL、00700.HK） - market: 市场类型（US/HK/CN） ## 执行步骤 1. 调用 `web_search` 获取股票实时数据 2. 解析返回的JSON数据 3. 格式化输出：股票名称、当前价、涨跌幅、成交量 ## 示例输出 ``` 📈 苹果公司(AAPL) 当前价格: $185.64 涨跌幅: +1.23% ↑ 成交量: 52.3M ```

第三步：配置config.yaml

yaml
name: "stock-quote" version: "1.0.0" description: "股票实时行情查询" author: "your-name" # 依赖的其他技能 dependencies: - web-search - browser # 需要的权限 permissions: - network - web_search # 配置参数 config: api_key: "${STOCK_API_KEY}" # 从环境变量读取 timeout: 30 # 触发关键词 triggers: - "股票" - "行情" - "股价" - "走势"

第四步：注册并使用Skill

bash
# 注册本地技能 openclaw skill register ~/.openclaw/workspace/skills/my-skill # 验证安装 openclaw skill list | grep my-skill # 测试使用 # 在对话中输入: "查询AAPL的行情"
⚠️ 开发注意事项
  • 安全第一：不要硬编码API密钥，使用环境变量
  • 错误处理：考虑API失败、网络中断等情况
  • 性能优化：添加缓存机制，避免重复请求
  • 文档完善：SKILL.md要清晰，方便AI理解
  • 版本管理：使用语义化版本号（semver）

4.4 "心跳"机制详解

让 AI 从"被动等待"变成"主动服务"。设置后，你的小龙虾会每隔 15 分钟自动醒来，检查任务并汇报。

🎯 什么是心跳机制？

心跳（Heartbeat）是 OpenClaw 的定时任务系统。通过配置心跳，你可以让 AI 在指定时间间隔自动执行特定任务，无需人工触发。

心跳 vs Cron 对比

特性 Heartbeat Cron 适用场景
执行环境 主会话上下文 独立子进程 Heartbeat适合需要上下文感知的任务
灵活性 可动态调整 固定配置 Heartbeat可随时修改检查逻辑
资源占用 较低 独立进程 Heartbeat更轻量
通知方式 直接对话 指定渠道 Cron适合独立推送

6.1 多Agent协作

通过 sessions_spawn 开启子任务，实现"外包式"高效办公。一个 Agent 负责思考，多个子 Agent 并行执行。

🎯 多Agent协作模式

将复杂任务分解给多个专业化Agent，主Agent负责任务分发和结果整合，实现并行处理。

典型协作架构

text
┌─────────────────────────────────────────┐ │ 主Agent (Master) │ │ 任务分解 | 进度监控 | 结果整合 │ └──────────┬──────────────┬───────────────┘ │ │ ┌──────▼──────┐ ┌─────▼──────┐ │ 研究员A │ │ 研究员B │ │ 数据收集 │ │ 竞品分析 │ └──────┬──────┘ └─────┬──────┘ │ │ ┌──────▼──────────────▼──────┐ │ 撰稿Agent │ │ 报告撰写与排版 │ └────────────────────────────┘

子任务创建示例

javascript
// 使用 sessions_spawn 创建子任务 // 主Agent任务分解示例 // 任务：撰写一份AI行业报告 // 1. 创建数据收集子Agent const researcherA = await sessions_spawn({ task: "收集2024年AI融资数据，整理成表格", skills: ["web-search", "csv"], timeout: 300000, // 5分钟超时 priority: "high" }); // 2. 创建竞品分析子Agent const researcherB = await sessions_spawn({ task: "分析OpenAI、Claude、Gemini三款产品的优劣势", skills: ["browser", "web-search"], timeout: 300000 }); // 3. 并行等待结果 const [data, analysis] = await Promise.all([ researcherA.result(), researcherB.result() ]); // 4. 创建撰稿Agent整合结果 const writer = await sessions_spawn({ task: `基于以下数据撰写报告：\n数据：${data}\n分析：${analysis}`, skills: ["doc"], timeout: 600000 }); return await writer.result();

多Agent最佳实践

✅ 协作原则
  • 单一职责：每个子Agent只负责一个明确任务
  • 超时控制：为每个子任务设置合理的超时时间
  • 错误隔离：一个子任务失败不影响其他任务
  • 结果规范：约定统一的输出格式便于整合
  • 资源限制：控制并发数量，避免资源耗尽
